# MDeX v1.4.0 (macOS · Windows · Linux · Полностью автономно · Tauri v2)

> **MDeX** · произносится «эм-декс» (/ˌemˈdɛks/) — буква M, за которой следует «dex», два слога.

Автономный (offline-first) редактор и просмотрщик Markdown для использования **в изолированных сетях / интрасетях / без подключения к интернету**. Каждый файл обрабатывается локально — **без сети, без загрузки на сервер, без облачной синхронизации, без рекламы, без телеметрии, без передачи данных**.

- Единый автономный HTML-фронтенд (без Vue / React); Tauri v2 предоставляет только нативную оболочку (окна, меню, диалоги выбора файлов).
- **Нулевые сетевые запросы во время выполнения**: `marked` / `KaTeX` / `highlight.js` / `DOMPurify` / `mermaid` / `jsPDF` / `html2canvas-pro` / `turndown` / `@retorquere/bibtex-parser` и все шрифты KaTeX woff2 встроены / закодированы в base64 в единый `index.html`.
- Поддерживаются `.md` / `.markdown` / `.html`; приложение можно назначить обработчиком `.md` по умолчанию — открывайте двойным щелчком.

## ✨ Возможности

- **Многовкладочность + многооконность**: открывайте несколько файлов одновременно; несохранённые изменения помечаются точкой, при закрытии запрашивается подтверждение; закройте вкладку средним щелчком. Двойной щелчок по `.md` открывает собственное окно (по одному файлу на окно); двойной щелчок по уже открытому файлу **переводит фокус на это окно**, а не открывает его заново.
- **Живое разделённое предпросмотр**: перетаскивайте разделитель для изменения размера; кнопка на панели инструментов циклически переключает режимы «Разделённый / Редактор / Предпросмотр».
- **Переход по щелчку**: щёлкните в редакторе, чтобы прокрутить предпросмотр; щёлкните в предпросмотре, чтобы перевести курсор в редакторе.
- **Поиск и замена**: найти, заменить одно или все совпадения, с подсчётом количества.
- **Математика**: строчные `$…$` и блочные `$$…$$` (а также `\(...\)`, `\[...\]`), отображаются через KaTeX; длинные формулы переносятся по операторам или автоматически уменьшаются.
- **Подсветка кода**: язык определяется автоматически; подсветка в больших документах лениво выполняется по видимой области для плавности.
- **Диаграммы Mermaid**: блок ` ```mermaid ` отображается как блок-схема / последовательность / классы / состояния / диаграмма Ганта / круговая и т. д.; щёлкните по диаграмме, чтобы открыть отдельное окно просмотра (масштаб / панорама / полноэкранный режим), которое обновляется в реальном времени при редактировании.
- **Изображения**: вставить / перетащить / выбрать — сохраняются в папку `<имяфайла>_images/` рядом с документом с чистой относительной ссылкой (без встроенного base64); черновики используют временную папку, мигрирующую при сохранении; «Сохранить как» переносит изображения в цель; по умолчанию центрируются.
- **Масштаб шрифта**: масштабируйте шрифты редактора и предпросмотра независимо (элементы управления −/процент/+, или `⌘/Ctrl + =/−/0`); сохраняется между перезапусками.
- **Таблицы**: таблицы GFM; узкие таблицы центрируются по содержимому, широкие прокручиваются по горизонтали без обрезки.
- **Цитирование (BibTeX)**: синтаксис `[@key]` / `\cite{key}`, числовой стиль; в конце формируется список ссылок с двусторонними переходами между в текстовой `[n]` и записью; поддерживается встроенный блок ` ```bibtex ` или отдельно загружаемый `.bib`.
- **Поддержка HTML**: открывайте файлы `.html` для отображения; конвертируйте между HTML и Markdown.
- **Тема / язык**: тёмная / светлая, **17 языков интерфейса** (中文, English, Français, Deutsch, Русский, Italiano, 日本語, 한국어, Español, Português, العربية, हिन्दी, ਪੰਜਾਬੀ, Tiếng Việt, Bahasa Indonesia, اردو, Монгол (Кирилл) — арабский и урду автоматически справа налево).
- **Автосохранение черновика**: содержимое периодически сохраняется и восстанавливается после непредвиденного закрытия / сбоя.
- **Подсчёт слов**: в строке состояния отображаются символы / строки / слова в реальном времени, а также текущие строка и столбец.
- **Перетаскивание**: перетащите файл `.md` на окно, чтобы открыть его; перетащите изображение, чтобы вставить его.
- **Экспорт**: сохранение как Markdown / HTML / PDF (векторный) / LaTeX.
- **Цвет текста**: палитра на панели обёртывает выделение в `<span style="color:…">`.
- **Назад / Вперёд**: единая история по документам и позициям курсора; кнопки ◀ ▶, `Alt+←/→`.
- **Переход по ссылкам**: клик по ссылке в предпросмотре открывает цель в новой вкладке (http — в системном браузере); текущий документ не заменяется.

## 📝 Краткая шпаргалка

**Markdown**: заголовки `# / ## / ###`, полужирный `**текст**`, курсив `*текст*`, зачёркивание `~~текст~~`, строчный код `` `код` ``, блоки кода (тройные обратные кавычки с указанием языка по желанию), цитата `> текст`, списки `- / 1.`, список задач `- [ ] / - [x]`, ссылка `[текст](url)`, изображение `[alt](url)`, разделитель `---`, таблицы `| A | B |`.

**Математика**: строчно `$E = mc^2$`; блочно `$$\int_0^1 x\,dx$$` (может занимать несколько строк). Используется синтаксис LaTeX, отображается через KaTeX; `$` внутри блоков кода не считается разделителем формул. Поддерживаются окружения `align` / `aligned`, матрицы, `cases` и другие распространённые.

**Цитирование**: напишите в тексте `[@key]` или `[@a; @b]` (совместимый с LaTeX `\cite{key}`), вставьте библиотеку через блок ` ```bibtex ` или загрузите `.bib` кнопкой «Ссылки». В конце формируется список ссылок; `[n]` в тексте кликабельны.

## 📤 Экспорт (Сохранить как)

Нажмите «Сохранить как» и выберите формат:

- **Markdown (.md)**: сохранить исходник и обновить имя / путь текущей вкладки.
- **HTML (.html)**: автономный HTML со встроенным CSS + подсветкой кода; формулы сохранены как литерал `$…$` и автоматически отображаются встроенным KaTeX.
- **PDF векторный**: системный диалог печати, векторный вывод, чёткий при любом масштабе. Выберите «Сохранить как PDF».
- **LaTeX (.tex)**: преобразуется в компилируемый исходник `.tex` (с documentclass и пакетами; формулы сохраняются как есть). Экспортируется копия.

## 🔒 Автономность и безопасность

- **Нулевые сетевые запросы во время выполнения.** Артефакт сборки `dist/index.html` самопроверяется: отсутствуют внешние ссылки `src=` / `href=` / `url()` / `@import`.
- Строгий CSP (только локальный IPC, без WAN); все файлы читаются и пишутся локально, ничего не передаётся.
- Проверка: отключите Wi-Fi / отсоедините кабель и запустите — формулы, изображения, подсветка кода и Mermaid продолжают работать.
- В `dist/index.html` всё ещё присутствует около десятка строк `https://github.com/…`; все они находятся внутри **лицензионных / исходных комментариев** библиотек `marked` / `highlight.js` и т. п. — это простой текст, который **никогда не инициирует запрос**; оставлены без изменений из уважения к лицензиям open-source.

### Открытие неподписанного приложения на macOS (обход Gatekeeper)

Это приложение **не** имеет подписи разработчика / не нотаризовано (в автономных сценариях нотаризация онлайн обычно невозможна). На macOS 12+, **особенно macOS 26 (Tahoe)**, запуск прямо из `.dmg` — или свежескопированной сборки — завершается ошибкой **"MDeX.app is damaged and can't be opened."** Это Gatekeeper, а не реальное повреждение. Исправьте в Terminal:

1. **Сначала перетащите `MDeX.app` из `.dmg` в `/Applications`** — никогда не запускайте напрямую из dmg (это вызывает App Translocation и атрибут `com.apple.provenance`, настоящую причину «повреждено» на macOS 26).
2. Очистите атрибуты и переподпишите:
   ```bash
   xattr -cr /Applications/MDeX.app
   codesign --force --deep --sign - /Applications/MDeX.app
   ```
   > `com.apple.provenance` защищён SIP и **не может** быть удалён даже с `sudo`; переподпись сбрасывает подпись, чтобы Gatekeeper разрешил запуск. `spctl` всё равно сообщает `rejected` для ad-hoc подписи — это ожидаемо и **не** блокирует `open`.
3. Запустите через `open /Applications/MDeX.app` (или двойной клик). При первом запуске может появиться запрос — подтвердите через **Системные настройки → Конфиденциальность и безопасность → Открыть всё равно**, или кликните правой кнопкой по приложению → **Открыть**.

## 📁 Структура проекта

```
markdown/
├── app-shell.html          # frontend shell (HTML+CSS); app logic lives in src/app.js
├── src/
│   ├── app.js              # application logic (// @ts-check; inlined into dist by build-html.mjs)
│   └── globals.d.ts        # vendor / Window type declarations for type-checking
├── tsconfig.json           # type-check config (tsc --noEmit; no bundler)
├── tools/
│   ├── fetch-vendor.mjs    # one-time: download deps into vendor/ + integrity lock (online only here)
│   ├── build-html.mjs      # inline vendor + src/app.js into dist/index.html (KaTeX fonts → base64)
│   └── test-pure.mjs       # frontend pure-function tests (npm test)
├── dist/index.html         # build output: fully offline single file (Tauri frontendDist)
├── vendor/                 # download cache + integrity.json (.gitignore)
├── package.json            # @tauri-apps/cli + typescript(dev) + scripts
└── src-tauri/
    ├── Cargo.toml          # tauri 2 + dialog / single-instance + encoding_rs
    ├── build.rs            # tauri_build::build()
    ├── tauri.conf.json     # 1200×750 window, strict CSP, icons, .md association, menu hooks
    ├── capabilities/default.json
    ├── icons/              # full icon set from `cargo tauri icon`
    └── src/{main.rs, lib.rs}   # menus + file IO + multi-window routing + atomic write / file ownership
```

## 🎨 Кастомизация

| Версии зависимостей | `VERSIONS` в начале `tools/fetch-vendor.mjs` (затем `npm run fetch -- --force`) |
